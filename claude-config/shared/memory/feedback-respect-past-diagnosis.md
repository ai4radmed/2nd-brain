---
name: 사용자가 확정해 준 제약·판단은 재진단하지 말 것
description: 과거 스크린샷·설명으로 결론 난 사항은 메모리에 저장, 세션마다 다시 풀려 들지 말 것
type: feedback
---

사용자가 **"이미 확인했고 이건 막혀 있다"** 고 명시한 제약·판단은, 다음 세션에서도 **그대로 받아들일 것**. 재진단·대안 재제시로 사용자를 다시 설명하게 만들지 말 것.

**Why:** 한 사용자가 "Gmail 쓰기 도구 제약을 과거에 스크린샷으로 설명했는데 어시스턴트가 반복해서 헷갈려 한다"고 지적한 사례. 어시스턴트의 기본 편향("도움이 되려고 재시도")이 사용자의 과거 판단을 무효화하는 결과를 낳았음. 메모리 저장 누락 + 잘못된 워크플로우 문서 + 재진단 선호가 복합 작용.

**How to apply:**

1. 사용자가 "이미 확인했다", "스크린샷으로 설명한 적 있다", "이건 구조적으로 안 된다" 같은 발언을 하면 → **즉시 project 또는 feedback 메모리로 저장**. 저장 없이 "알겠습니다" 하지 말 것
2. 제약에 대한 우회책 제시는 **사용자가 먼저 꺼냈을 때만**. 선제적으로 "이렇게 하면 되지 않을까요" 를 남발하지 말 것
3. 워크플로우 문서에 틀린 진단이 남아 있으면 **즉시 수정**. 문서와 메모리가 일치하게 유지
4. 매 세션 시작 시 MEMORY.md 와 관련 메모리 파일을 먼저 읽어 사용자의 과거 결정을 존중
