# 자동 우선·주간 감사 (Automate-First, Weekly-Audit)

2nd-brain 정보 편입의 운영 원칙. **정보를 시스템에 들이는 일(capture → parse → 요약·분류 → PARA 배치)은 건별 사람 승인 없이 자동**으로 하고, **품질 보증은 주 1회 고정 시간의 감사**로 모은다.

## 왜

- **건별 승인은 안 한다** — 매 문서마다 사람이 확인하면 자동화의 의미가 사라지고, 쌓이면 밀린다.
- **그렇다고 무검증도 아니다** — 자동 분류·변환은 오차가 있다(특히 PARA 좌표, 형식 변환 손실).
- 절충: **낙관적으로 자동 배치 + 의심분 플래그 + 주간 감사로 사후 교정**. 사람 시간은 주 1회 슬롯에만, 오차는 유실이 아니라 *지연 교정*.

GTD/BASB 의 주간 리뷰 주기와 같은 리듬이다.

## 두 모드

| 평시 (건별, 자동) | 주 1회 (사람) |
|---|---|
| capture → parse → 요약·분류 → **낙관적 PARA 배치 + 동반 노트** | 그 주 자동 처리분 **비교 감사** — 좌표 교정·변환 품질 점검 |

## 플래그 규약 (frontmatter)

자동 처리분에 신호를 달아, 감사가 *의심스러운 것 우선* 보게 한다 (전수 아님):

- `para_review: pending` — 낙관적 PARA 배치, 좌표 미확정.
- `parse_confidence: low` — 변환 손실 의심(예: 형식 변환 결과가 비정상적으로 짧거나, 이중 파싱 경로의 결과가 어긋남).

감사 시: 좌표 적정 → 플래그 제거 / 부적정 → 노트·원본 동반 이동 + 출처 경로 갱신 + 플래그 제거.

## 주간 감사 절차

1. `para_review: pending` · `parse_confidence: low` 항목 수집.
2. 원본 ↔ 동반 노트 비교 — PARA 좌표·변환 품질 확인.
3. 교정 후 플래그 해제. 저품질 변환은 대체 경로로 재처리.

## 스케줄

주 1회 고정 슬롯(예: 일요일). 알림은 가벼운 자동(cron)으로, *감사 작업*은 사람이 검토 세션에서.

> 이 원칙은 어떤 자동 편입 파이프라인(메일 라벨 처리·문서 파서·brainify 등)에도 동일하게 적용한다 — 건별 승인 대신 *낙관 자동 + 플래그 + 주간 감사*.
